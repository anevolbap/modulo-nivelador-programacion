Clase 2: condicionales y bucles

Decidir y repetir

Hasta ahora supimos guardar valores y aplicar funciones. Pero los programas hacían siempre lo mismo.

Hoy le damos al programa dos capacidades nuevas:

  • decidir: ejecutar (o no) un bloque según una condición. Condicionales: if, else.
  • repetir: ejecutar un bloque varias veces. Bucles: for, while.

Volviendo a la analogía: en un idioma decimos “si llueve, llevo paraguas” (condicional) o “leí cada página del libro” (bucle). Ya tenemos los sustantivos y los verbos, ahora agregamos las conjunciones.

~5 min. Conectar con clase 1: variables (sustantivos) + funciones (verbos) + ahora condicionales y bucles.

if: ejecutar solo si

if (condicion) {
  # se ejecuta solo si condicion es TRUE
}

Ejemplo:

nota <- 7

if (nota >= 6) {
  print("aprobado")
}

condicion es una expresión que evalúa a TRUE o FALSE (un logical).

if / else: dos caminos

if (condicion) {
  # rama TRUE
} else {
  # rama FALSE
}
nota <- 4

if (nota >= 6) {
  print("aprobado")
} else {
  print("desaprobado")
}

if / else if / else: varios casos

if (x > 0) {
  print("positivo")
} else if (x < 0) {
  print("negativo")
} else {
  print("cero")
}

Las condiciones se evalúan de arriba hacia abajo y se ejecuta la primera que sea verdadera. Si ninguna lo es, se ejecuta el else (si existe).

Operadores en condiciones

Comparación: ==, !=, <, <=, >, >=.

Lógicos: & (y), | (o), ! (negación).

edad <- 20

if (edad >= 18 & edad < 65) {
  print("adulto en edad activa")
}

Recordar que == es comparación, = es asignación. Confundirlos es el error tipográfico más común en R.

Trampa frecuente: == vs =

if (x == 5) ...   # OK: compara x con 5
if (x = 5) ...    # ERROR de sintaxis

== pregunta. = declara. Si la palabra “if” no funciona como esperaban, casi siempre es esto.

ifelse(): condicional vectorial

Cuando queremos aplicar el mismo if/else a cada elemento de un vector, hay un atajo:

notas   <- c(7, 4, 9, 5, 6)
estados <- ifelse(notas >= 6, "aprobado", "desaprobado")
estados
# [1] "aprobado"    "desaprobado" "aprobado"    "desaprobado" "aprobado"

ifelse(test, valor_si_TRUE, valor_si_FALSE) recorre el vector y elige.

¿Qué es un bucle?

Un bucle ejecuta el mismo bloque de código varias veces. Sirve, entre otras cosas, para

  • aplicar una transformación a cada elemento de un vector;
  • repetir un experimento aleatorio muchas veces;
  • iterar hasta que un proceso converja (bisección, descenso de gradiente).

for: sintaxis

for (variable in coleccion) {
  # se ejecuta una vez por cada elemento de coleccion
}

En cada iteración, variable toma un valor distinto de coleccion.

for (i in 1:5) {
  print(i)
}
# 1 2 3 4 5

for: recorrer un vector

Por elemento:

nombres <- c("Ana", "Beto", "Cami")
for (n in nombres) {
  print(paste("Hola,", n))
}

Por índice (cuando necesitamos saber la posición):

for (i in seq_along(nombres)) {
  print(paste(i, nombres[i]))
}

Tip: seq_along(x) es más seguro que 1:length(x). Si x tiene largo cero, 1:length(x) devuelve c(1, 0) y el bucle se ejecuta dos veces sin sentido.

for: acumular en un vector

Patrón estándar: pre-asignar el vector y rellenarlo.

x <- 1:10
y <- vector("numeric", length(x))   # pre-asignar

for (i in seq_along(x)) {
  y[i] <- x[i]^2
}

Evitar crecer el vector dentro del bucle con y <- c(y, x[i]^2): cada paso copia todo, y se vuelve lento muy rápido.

for anidado

Un bucle dentro de otro. Útil para recorrer estructuras 2D.

for (i in 1:3) {
  for (j in 1:3) {
    print(paste(i, j))
  }
}

while: sintaxis

while (condicion) {
  # se ejecuta mientras condicion sea TRUE
}

Ejemplo: duplicar hasta superar 1000.

x <- 1

while (x < 1000) {
  x <- x * 2
}
x   # 1024

A diferencia del for, no sabemos de antemano cuántas iteraciones va a hacer.

El peligro del bucle infinito

La condición del while tiene que poder volverse FALSE en algún momento. Si no, el programa no termina.

i <- 1

while (i > 0) {
  print(i)
  # i <- i - 1   # FALTA esta linea
}

En RStudio se corta con Esc o el botón rojo de stop en la consola.

¿for o while?

  • for: cuando sabemos cuántas iteraciones (recorrer un vector, repetir N veces).
  • while: cuando NO sabemos cuántas, sino que iteramos hasta que algo pase (convergencia, búsqueda, condición de corte).

En la duda, for es más seguro porque siempre termina.

break y next

  • break: salir del bucle inmediatamente.
  • next: saltear la iteración actual y seguir con la próxima.
for (i in 1:10) {
  if (i == 5) break          # corta el bucle
  if (i %% 2 == 0) next      # saltea pares
  print(i)
}
# 1 3

Buena práctica: comentar el porqué, no el qué

Si los nombres son buenos, el código ya cuenta qué hace. Los comentarios son para explicar por qué ese código está así: una decisión, una excepción, un dato externo.

# parafrasea: no agrega informacion
x <- 5  # asigno 5 a x

# explica el porqué: agrega información
edad_minima_aprobado <- 18  # mayoria de edad civil en Argentina (Cod. Civ. art 25)

Reglas:

  • Si tenés que parafrasear el código, mejorá el nombre.
  • Si una decisión no es obvia, escribí en una línea por qué.
  • Borrar comentarios desactualizados es tan importante como escribirlos.

Comprensión: ¿quedó claro?

Marcá la opción correcta.

  1. ¿Qué imprime este código?

    x <- 5
if (x > 0) print("positivo") else print("no positivo")
    1. "positivo"
    2. "no positivo"
    3. da error
    4. no imprime nada

  2. ¿Cuántos números imprime este bucle?

    for (i in 1:10) {
  if (i == 5) break
  print(i)
}
    1. 4
    2. 5
    3. 9
    4. 10

  3. ¿Cuál estructura es más adecuada cuando NO sabés cuántas iteraciones hay que hacer, sino que repetís hasta que ocurra algo?

    1. for
    2. while
    3. if
    4. replicate

  4. ¿Qué problema tiene este código?

    i <- 1
while (i < 10) {
  print(i)
}
    1. Sintaxis inválida.
    2. Imprime 9 veces y termina.
    3. Es un bucle infinito porque i nunca cambia.
    4. Imprime una sola vez.

Respuestas: 1-a, 2-a (imprime 1..4 y luego entra en la iteración i=5 donde rompe sin imprimir), 3-b, 4-c.

Apéndice: alternativas a for

R tiene formas más cortas de aplicar una función a cada elemento.

# repetir N veces una expresión
replicate(5, sample(1:6, 1))

# aplicar una función a cada elemento, devolver lista
lapply(c(1, 2, 3), function(x) x^2)

# aplicar una función a cada elemento, devolver vector si se puede
sapply(c(1, 2, 3), function(x) x^2)

No reemplazan al for, pero suelen leerse mejor cuando la tarea es “para cada elemento, hacer X”. Las vamos a usar más adelante.
