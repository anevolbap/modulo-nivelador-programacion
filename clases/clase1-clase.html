Curso Nivelador - Programación en R

Clase 1: tipos de datos, estructuras y funciones

Programar es como hablar un idioma

Aprender un lenguaje de programación se parece a aprender un idioma.

  • Los sustantivos son las variables: las cosas con las que trabajamos (un número, un nombre, una tabla).
  • Los verbos son las funciones: las acciones que ejecutamos sobre esas cosas (sumar, contar, ordenar).
  • La gramática es la sintaxis: cómo se combinan las palabras para que tengan sentido.
  • Los errores de tipeo existen también: si decimos algo mal, el otro lado no entiende.

Cada idioma tiene sus particularidades. En español el adjetivo va después del sustantivo, en inglés antes. R también tiene las suyas: el primer elemento de un vector es el 1, no el 0; la asignación se escribe <-. Son convenciones que se aprenden con el uso.

~5 min. Pedir un ejemplo de oración y traducirla al “verbo + objetos”.

¿Qué es R?

R es un lenguaje de programación y un entorno de cómputo orientado a estadística y análisis de datos.

  • Open source y gratuito.
  • Comunidad enorme: paquetes para casi cualquier tarea (CRAN, Bioconductor, GitHub).
  • Pensado para trabajar con datos: vectores, tablas, gráficos.
  • Lo van a usar mucho a lo largo del programa.

¿Y RStudio?

RStudio es un IDE (entorno integrado de desarrollo): un programa que envuelve a R y nos hace la vida más fácil.

Cuatro paneles típicos:

  1. Editor (arriba-izquierda): donde escribimos los scripts.
  2. Consola (abajo-izquierda): donde se ejecutan las instrucciones.
  3. Environment (arriba-derecha): qué variables tenemos en memoria.
  4. Files / Plots / Help (abajo-derecha): explorador, gráficos y ayuda.

Atajos útiles: Ctrl+Enter envía la línea actual a la consola; Ctrl+L limpia la consola.

Asignación: guardar un valor

Lo primero que vamos a hacer es guardar valores en variables.

x <- 10
nombre <- "Ana"
aprobado <- TRUE

<- es el operador de asignación: “el valor de la derecha va a la variable de la izquierda”. Una vez asignada, la variable se reutiliza:

y <- x + 5     # 15
nombre         # "Ana"

Reglas para nombres: empezar con letra, contener solo letras, números, _ y ., sin espacios y sin palabras reservadas (if, for, function, …).

Mostrar en vivo cómo aparece la variable en el panel Environment.

Cuatro tipos atómicos

Los tipos atómicos son los ladrillos básicos. Hay cuatro que vamos a usar todo el tiempo.

Tipo Para qué sirve Ejemplo
logical verdadero o falso TRUE, FALSE
double números reales 3.14, -2
integer números enteros 5L, 0L
character texto "hola", 'a'

class(x) nos dice de qué tipo es x. (Para double, class devuelve "numeric"; typeof devuelve "double").

logical

Toma uno de dos valores: TRUE o FALSE. Aparecen como resultado de comparaciones.

10 == 20       # FALSE
sin(pi/2) == 1 # TRUE
class(TRUE)    # "logical"

Operadores:

  • lógicos: & (y), | (o), ! (no);
  • de comparación: == (igual), != (distinto), <, <=, >, >=.

double y integer

double es el tipo numérico por defecto, sirve para reales.

radio <- 10
area  <- pi * radio^2
class(area)        # "numeric"

integer se usa para enteros. Para que R lo trate como entero hay que poner una L al final.

class(40)    # "numeric"
class(40L)   # "integer"

En la práctica, salvo que importe la performance o la interoperación con otros sistemas, alcanza con double.

character

Texto: símbolos, letras, palabras o frases dentro de comillas (dobles o simples).

saludo <- "Hola, mundo"
class(saludo)             # "character"
nchar(saludo)             # 11

Funciones útiles: paste, paste0, nchar, toupper, tolower, substr.

Coerción de tipos

Cuando mezclamos tipos, R toma una decisión por nosotros y convierte todo al tipo más “general”.

Reglas, de menor a mayor: logical → integer → double → character.

c(TRUE, 1L)         # integer:   1 1
c(TRUE, 1L, 1.5)    # double:    1.0 1.0 1.5
c(TRUE, 1, "a")     # character: "TRUE" "1" "a"

Es importante saberlo porque a veces lo causamos sin querer y los resultados se vuelven raros.

Vectores: lo mismo, varias veces

Un vector es una lista ordenada de valores del mismo tipo.

notas  <- c(7, 8, 5, 9, 6)
class(notas)             # "numeric"
length(notas)            # 5
notas[1]                 # 7
notas[c(2, 4)]           # 8 9
notas[notas > 6]         # 7 8 9

Si mezclamos tipos, R coerciona (slide anterior).

Listas: cosas distintas, juntas

Una lista combina elementos de distintos tipos y largos.

alumno <- list(
  nombre   = "Ana",
  edad     = 23L,
  notas    = c(7, 8, 9),
  aprobado = TRUE
)

Acceso por nombre o por posición:

alumno$nombre        # "Ana"
alumno[["notas"]]    # c(7, 8, 9)
alumno[[2]]          # 23

data.frame: la “tabla” de R

Un data.frame es una tabla: las filas son observaciones, las columnas son variables.

Por dentro, es una lista donde cada elemento es una columna y todas las columnas tienen la misma longitud.

estudiantes <- data.frame(
  nombre   = c("Ana", "Beto", "Cami"),
  nota     = c(7, 4, 9),
  aprobado = c(TRUE, FALSE, TRUE)
)
estudiantes$nota         # 7 4 9
estudiantes[1, ]         # primera fila
estudiantes[, "nombre"]  # columna por nombre

Va a ser la estructura central de la materia.

Atómicos vs compuestos

Estructura ¿Mismo tipo? ¿Misma longitud? Forma
vector sí (no aplica) unidimensional
lista no no “cajón con etiquetas”
data.frame por columna sí tabla

Funciones: nuestros propios verbos

Una función es un bloque de código con un nombre, que recibe argumentos y devuelve una salida.

trasladar <- function(x, a) {
  resultado <- x + a
  return(resultado)
}

trasladar(1, 5)      # 6
trasladar(1:10, 5)   # 6 7 8 ... 15

Argumentos y valores por defecto

Argumentos pueden pasarse por posición o por nombre:

trasladar(x = 1, a = 5)
trasladar(a = 5, x = 1)   # equivalente

También se pueden definir valores por defecto:

trasladar <- function(x, a = 1) {
  x + a
}

trasladar(10)      # 11, usa a = 1
trasladar(10, 3)   # 13

Buena práctica: nombres descriptivos

El nombre de una variable es la primera línea de documentación del código. Si los nombres son buenos, casi no hace falta comentar.

# poco claro
x  <- 25
df <- read.csv("datos.csv")
n  <- 100

# claro
edad_paciente   <- 25
pacientes       <- read.csv("datos.csv")
n_simulaciones  <- 100

Reglas prácticas:

  • snake_case (palabras separadas por _).
  • Singular para escalares, plural para vectores: edad vs edades.
  • Evitar nombres que pisan funciones de R: c, t, T, F, data, df, mean.

Comprensión: ¿quedó claro?

Marcá la opción correcta.

  1. ¿Cuál de estos nombres de variable es válido en R?
    1. 2x
    2. mi.variable
    3. mi variable
    4. for
  2. ¿Qué devuelve class(3.14)?
    1. "integer"
    2. "double"
    3. "numeric"
    4. "real"
  3. ¿Cuál es la diferencia clave entre una lista y un data.frame?
    1. Las listas no se pueden imprimir.
    2. En un data.frame todas las columnas deben tener la misma longitud.
    3. Un data.frame solo admite números.
    4. No hay diferencia.
  4. ¿Qué devuelve class(TRUE & FALSE)?
    1. "boolean"
    2. "logical"
    3. "TRUE"
    4. "vector"

Respuestas: 1-b, 2-c, 3-b, 4-b. Discutir las trampas: for es palabra reservada; class de un double devuelve "numeric" (no "double"), aunque typeof sí los distingue.

Apéndice: = vs <-

Para asignar en R hay dos operadores: <- y =. La mayoría del tiempo son intercambiables.

x <- 10
y =  20

La diferencia aparece cuando hay una llamada a función de por medio.

sum(x = 1:10)    # x es el nombre del argumento, no se asigna nada
sum(x <- 1:10)   # x queda asignada en el environment global

Convención que vamos a seguir:

  • Usar <- para asignar variables.
  • Usar = solo para pasar argumentos nombrados a una función.

Es una distinción menor, pero ayuda a leer el código sin ambigüedad.
