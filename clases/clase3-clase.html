Curso Nivelador - Programación en R

Clase 3: estructuras compuestas (lista, matriz, data.frame, factor)

De un valor a una colección

En clase 1 vimos los tipos atómicos: logical, double, integer, character. Cada variable guardaba un valor (o un vector de valores del mismo tipo).

Hoy vamos a ver cómo guardar colecciones más ricas:

  • lista: cosas distintas, juntas, con nombre.
  • matriz: tabla rectangular de valores del mismo tipo.
  • data.frame: tabla rectangular donde cada columna puede tener su tipo.
  • factor: variable categórica con un conjunto fijo de valores posibles.

~5 min. Recordar la analogía: hasta ahora teníamos sustantivos sueltos, ahora armamos “frases compuestas”.

Un mapa de las estructuras

Estructura Dimensiones ¿Mismo tipo?
vector 1 sí
matriz 2 sí
array n sí
lista 1 no
data.frame 2 por columna
factor 1 sí (entero)

vector y matriz son rígidos: todo del mismo tipo. lista y data.frame son flexibles.

Vectores con nombre (puente)

Un vector puede tener nombres en sus posiciones:

edades <- c(ana = 23, beto = 31, cami = 19)
edades["beto"]    # 31
names(edades)     # "ana"  "beto" "cami"

Esto ya empieza a parecerse a una lista. La diferencia: los elementos siguen teniendo que ser del mismo tipo.

Listas: motivación

Cuando una “cosa” tiene piezas heterogéneas, necesitamos una lista.

Por ejemplo, un alumno tiene un nombre (string), una edad (entero), un vector de notas y un booleano de aprobado.

alumno <- list(
  nombre   = "Ana",
  edad     = 23L,
  notas    = c(7, 8, 9),
  aprobado = TRUE
)

Una lista es como un cajón con etiquetas: cada compartimento puede contener algo distinto.

Listas: tres formas de acceder

alumno$nombre        # "Ana"          (por nombre, atajo)
alumno[["nombre"]]   # "Ana"          (por nombre, forma larga)
alumno[[1]]          # "Ana"          (por posición)

Y la forma “que parece igual pero no”:

alumno["nombre"]     # lista de un elemento, no el string
class(alumno["nombre"])    # "list"
class(alumno[["nombre"]])  # "character"

Regla mental: [[ ]] saca el contenido; [ ] saca un sub-cajón.

Esta es LA trampa de listas. Insistir con un par de ejemplos.

Listas anidadas

Una lista puede contener otras listas. Útil cuando los datos tienen jerarquía.

curso <- list(
  nombre  = "Nivelador",
  alumnos = list(
    list(nombre = "Ana",  nota = 7),
    list(nombre = "Beto", nota = 5)
  )
)

curso$alumnos[[1]]$nombre    # "Ana"

Matrices

Una matriz es una tabla rectangular de valores del mismo tipo.

M <- matrix(1:12, nrow = 3, ncol = 4)
M
#      [,1] [,2] [,3] [,4]
# [1,]    1    4    7   10
# [2,]    2    5    8   11
# [3,]    3    6    9   12

Por defecto se rellena por columna. Para rellenar por fila:

matrix(1:12, nrow = 3, byrow = TRUE)

Acceso en matrices

M[fila, columna]. Vacío en una posición = “todas”.

M[1, 2]    # 4   (fila 1, columna 2)
M[1, ]     # 1 4 7 10  (fila 1 entera)
M[, 2]     # 4 5 6     (columna 2 entera)
M[1:2, ]   # primeras dos filas

dim(M)     # 3 4
nrow(M)    # 3
ncol(M)    # 4
t(M)       # traspuesta
M %*% t(M) # producto matricial

data.frame: la tabla de R

Un data.frame es una tabla donde:

  • las filas son observaciones,
  • las columnas son variables,
  • cada columna puede tener su propio tipo.

Por dentro es una lista donde cada elemento es una columna y todas tienen la misma longitud.

estudiantes <- data.frame(
  nombre   = c("Ana", "Beto", "Cami"),
  nota     = c(7, 4, 9),
  aprobado = c(TRUE, FALSE, TRUE)
)

Acceso a un data.frame

estudiantes$nota              # columna como vector
estudiantes[, "nombre"]       # columna por nombre
estudiantes[1, ]              # primera fila
estudiantes[2, "nota"]        # celda fila 2, columna "nota"
estudiantes[, c("nombre", "nota")]   # varias columnas

$ es el atajo más usado para columnas. [fila, columna] es general.

Filtrar filas con condición lógica

estudiantes[estudiantes$nota >= 6, ]
#   nombre nota aprobado
# 1    Ana    7     TRUE
# 3   Cami    9     TRUE

La idea: estudiantes$nota >= 6 es un vector logical, y se usa como índice de filas.

estudiantes$nota >= 6   # TRUE FALSE TRUE

Modificar un data.frame

Agregar columna:

estudiantes$bonus <- estudiantes$nota + 1

Quitar columna (asignar NULL):

estudiantes$bonus <- NULL

Agregar fila con rbind:

nuevo <- data.frame(nombre = "Dani", nota = 6, aprobado = TRUE)
estudiantes <- rbind(estudiantes, nuevo)

Inspección rápida

Para ver qué hay adentro sin imprimir todo:

head(estudiantes)    # primeras 6 filas
tail(estudiantes)    # últimas 6 filas
dim(estudiantes)     # filas, columnas
str(estudiantes)     # estructura por columna
summary(estudiantes) # resumen estadístico
colnames(estudiantes)

str() es el más útil al recibir un dataset nuevo: en una mirada se ve cuántas filas, qué columnas y de qué tipo.

Cargar datos de un archivo

Para CSV (lo más común):

df <- read.csv("datos/bikeshare.csv")

Argumentos útiles:

  • header = TRUE (por defecto): primera fila son nombres de columnas.
  • sep = "," (por defecto): separador. En español a veces es ";".
  • stringsAsFactors = FALSE (por defecto desde R 4.0): los strings quedan como character, no factor.

Después suele convenir un str(df) o head(df) para confirmar que se cargó como esperábamos.

factor: variables categóricas

Un factor representa una variable que toma valores en un conjunto fijo (categorías).

nivel <- factor(c("alto", "bajo", "alto", "medio"))
nivel
# [1] alto  bajo  alto  medio
# Levels: alto bajo medio

levels(nivel)    # "alto" "bajo" "medio"

Por dentro, R guarda enteros y una tabla de niveles. Es eficiente y deja claro que los valores son categorías.

factor con orden

Si las categorías tienen un orden natural (“bajo” < “medio” < “alto”), se puede declarar:

nivel <- factor(
  c("alto", "bajo", "alto", "medio"),
  levels  = c("bajo", "medio", "alto"),
  ordered = TRUE
)

nivel[1] > nivel[2]   # TRUE: "alto" > "bajo"

Cuándo conviene: variables categóricas en modelos (lm, glm), ejes de gráficos, agrupaciones.
Cuándo evitar: para tratar como texto libre.

Para elegir estructura

Querés… Usá
una secuencia de valores del mismo tipo vector
una tabla rectangular numérica matrix
una “ficha” con campos heterogéneos list
una tabla de datos (mezcla de tipos por col.) data.frame
una variable categórica con valores fijos factor

Buena práctica: una función, una responsabilidad

Si una función “carga el CSV, le saca outliers, calcula la media y grafica”, es difícil de probar, reusar y debuggear.

Mejor partirla en cuatro funciones chicas:

# malo: hace de todo
analizar_pacientes <- function(archivo) { ... }

# bueno: una responsabilidad cada una
cargar_pacientes  <- function(archivo)        { ... }
filtrar_outliers  <- function(df, columna)    { ... }
calcular_resumen  <- function(df)             { ... }
graficar_resumen  <- function(resumen)        { ... }

Si cada pieza es chica, podés probarla por separado, reusarla en otro contexto y reemplazarla sin romper el resto.

Comprensión: ¿quedó claro?

Marcá la opción correcta.

  1. ¿Cuál es la diferencia entre mi_lista[1] y mi_lista[[1]]?

    1. No hay diferencia.
    2. [1] devuelve una lista de un elemento; [[1]] devuelve el elemento mismo.
    3. [[1]] da error si la lista tiene más de un elemento.
    4. [1] da error si la lista no tiene nombres.

  2. Si M <- matrix(1:6, nrow = 2), ¿qué devuelve M[, 2]?

    1. Un escalar.
    2. Una matriz de 2x1.
    3. Un vector de 2 elementos.
    4. Un error.

  3. Para cargar un archivo CSV en R conviene usar:

    1. load("archivo.csv")
    2. read.csv("archivo.csv")
    3. library("archivo.csv")
    4. import("archivo.csv")

  4. ¿Cuándo conviene usar un factor en lugar de un character?

    1. Cuando los datos son números.
    2. Cuando son categorías con un conjunto fijo de valores.
    3. Siempre, factor es más rápido.
    4. Nunca, factor está deprecado.

Respuestas

  1. b) [1] devuelve una sub-lista de un elemento, [[1]] devuelve el contenido. Atajo mental: [[ ]] saca el contenido, [ ] saca un sub-cajón.
  2. c) Vector de 2 elementos. Por defecto R “tira” la dimensión cuando queda una sola fila o columna (drop = TRUE).
  3. b) read.csv. load es para .RData, library carga paquetes.
  4. b) Para variables categóricas con un conjunto fijo de valores. Es lo que esperan los modelos de R y deja claro que no son texto libre.

Apéndice: una lista de data.frames

Cuando manejamos varios datasets a la vez, una lista de data.frames es muy práctica.

datasets <- list(
  norte = read.csv("datos/norte.csv"),
  sur   = read.csv("datos/sur.csv")
)

# aplicar una función a cada uno
sapply(datasets, nrow)
#  norte    sur
#    100    150

Combinarlos verticalmente:

todo <- do.call(rbind, datasets)

Esta combinación de listas + data.frames + sapply aparece todo el tiempo en análisis de datos.
