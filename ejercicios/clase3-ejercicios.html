Ejercicio 1: lista paciente

  1. Creá una lista paciente con: id (integer), edad (double), sintomas (vector de strings), tiene_diagnostico (logical) y presion_sistolica (double).

  2. Accedé al primer síntoma con $, con [[ ]] y con [ ]. Mostrá class() de cada acceso y explicá la diferencia.

  3. Reusá clasificar_presion (clase 2) para agregar un campo presion_categoria a la lista. Verificá.

TipSolución 
paciente <- list(
  id                = 1001L,
  edad              = 47.5,
  sintomas          = c("tos", "fiebre"),
  tiene_diagnostico = TRUE,
  presion_sistolica = 138
)

# b)
paciente$sintomas[1]            # "tos"  (character)
paciente[["sintomas"]][1]       # "tos"  (character)
paciente["sintomas"]            # lista de 1 elemento (list)

class(paciente$sintomas[1])         # "character"
class(paciente[["sintomas"]][1])    # "character"
class(paciente["sintomas"])         # "list"

# c)
paciente$presion_categoria <- clasificar_presion(paciente$presion_sistolica)
paciente$presion_categoria   # "hipertension 1"

Por qué [ ] devuelve lista: [ ] es el indexado “preserva estructura”: si tenés una lista, te devuelve una lista (eventualmente con varios elementos). [[ ]] extrae UN elemento. Esta es probablemente la trampa más común al trabajar con listas.

Ejercicio 4: matriz desde airquality

  1. Convertí las cuatro columnas numéricas de airquality (Ozone, Solar.R, Wind, Temp) en una matriz: M <- as.matrix(airquality[, 1:4]).

  2. Calculá la media por columna con colMeans(M, na.rm = TRUE).

  3. Calculá M_clean <- na.omit(M) y luego t(M_clean) %*% M_clean / nrow(M_clean). ¿Qué representa cada celda? Pista: el producto interno entre dos columnas, normalizado.

TipSolución 
M <- as.matrix(airquality[, 1:4])
class(M)   # "matrix" "array"
dim(M)     # 153 4

# b)
colMeans(M, na.rm = TRUE)
# Ozone: 42.13   Solar.R: 185.93   Wind: 9.96   Temp: 77.88

# c)
M_clean <- na.omit(M)
nrow(M_clean)   # 111 (descarto las 42 filas con al menos un NA)
G <- t(M_clean) %*% M_clean / nrow(M_clean)
G
# 4x4. La celda (i, j) es la media del producto x_i * x_j.
# La diagonal es la media de x_i^2 (segundo momento, no la varianza).

Por qué importa este producto: centrando las columnas (restando su media) antes de calcular t(X) %*% X / n, lo que sale es la matriz de covarianza. Sin centrar es el “momento de orden 2”. Es el cálculo base de muchos métodos: PCA, regresión, geometría de datos.

Ejercicio 5: operaciones con matrices

  1. Construí la matriz identidad de 5x5 con diag(5). Multiplicala por sí misma con %*%. Verificá que el resultado sigue siendo la identidad.

  2. Construí una matriz A de 3x3 con valores aleatorios (matrix(rnorm(9), 3, 3)). Calculá su traspuesta y verificá que t(t(A)) da A.

  3. Calculá A %*% t(A). ¿Por qué es simétrica?

TipSolución 
# a)
I5 <- diag(5)
all(I5 %*% I5 == I5)   # TRUE

# b)
set.seed(42)
A <- matrix(rnorm(9), 3, 3)
identical(t(t(A)), A)   # TRUE

# c)
S <- A %*% t(A)
all(S == t(S))   # TRUE  (es simetrica)

Por qué A %*% t(A) siempre es simétrica: llamemos B = A %*% t(A). Entonces t(B) = t(t(A)) %*% t(A) = A %*% t(A) = B. Más aún: es semi-definida positiva (todos sus autovalores son no negativos). Es la base de muchas factorizaciones útiles (Cholesky, SVD).

Detalle de igualdad: all(S == t(S)) puede fallar por punto flotante. Para comparar matrices con tolerancia: all(abs(S - t(S)) < 1e-10).

Ejercicio 7: medias_por_grupo

  1. Definí medias_por_grupo(df, columna_numerica, columna_grupo) que devuelva un data.frame con dos columnas, grupo y media. Pista: aggregate. Manejá NA con na.rm = TRUE.

  2. Aplicala a airquality: media de Ozone por Month.

  3. Aplicala a iris: media de Sepal.Length por Species.

TipSolución 
medias_por_grupo <- function(df, columna_numerica, columna_grupo) {
  formula <- as.formula(paste(columna_numerica, "~", columna_grupo))
  resumen <- aggregate(formula, data = df, FUN = mean, na.rm = TRUE)
  names(resumen) <- c("grupo", "media")
  resumen
}

# b)
medias_por_grupo(airquality, "Ozone", "Month")
# 5: ~24,  6: ~30,  7: ~59,  8: ~60,  9: ~31

# c)
medias_por_grupo(iris, "Sepal.Length", "Species")
# setosa: 5.01,  versicolor: 5.94,  virginica: 6.59

Por qué as.formula: aggregate espera una fórmula como Ozone ~ Month. Pero acá los nombres de columna vienen como strings. as.formula(paste(...)) arma la fórmula a partir de strings. Es el patrón estándar para escribir funciones que reciben nombres de columna como parámetros.

Ejercicio 8: top_grupo

  1. Definí top_grupo(resumen) que reciba la salida de medias_por_grupo y devuelva el grupo con la media más alta.

  2. Aplicala al resultado de 7b. ¿Qué mes tiene mayor ozono medio?

  3. Aplicala al resultado de 7c. ¿Qué especie tiene los sépalos más largos en promedio?

TipSolución 
top_grupo <- function(resumen) {
  resumen$grupo[which.max(resumen$media)]
}

# b)
top_grupo(medias_por_grupo(airquality, "Ozone", "Month"))   # 8 (agosto)

# c)
top_grupo(medias_por_grupo(iris, "Sepal.Length", "Species"))   # "virginica"

Caso no contemplado: si hay empate en la media máxima, which.max devuelve solo el primer índice. Si querés todos los empatados: resumen$grupo[resumen$media == max(resumen$media)].

Ejercicio 9: factor con orden

  1. Convertí mtcars$cyl en factor. Mostrá los niveles.

  2. Convertilo en factor ordenado c("4", "6", "8") con ordered = TRUE. Verificá que la comparación entre niveles funciona.

  3. Aplicá medias_por_grupo(mtcars, "mpg", "cyl"). ¿Confirma la intuición de que más cilindros implica menos mpg?

TipSolución 
# a)
cyl_f <- factor(mtcars$cyl)
levels(cyl_f)   # "4" "6" "8"

# b)
cyl_ord <- factor(mtcars$cyl, levels = c(4, 6, 8), ordered = TRUE)
cyl_ord[1] < cyl_ord[2]   # depende de los autos
"4" < "8"                 # FALSE como string (orden lexicografico)
factor(8, levels = c(4, 6, 8), ordered = TRUE) > factor(4, levels = c(4, 6, 8), ordered = TRUE)
# TRUE  (compara como ordinal)

# c)
medias_por_grupo(mtcars, "mpg", "cyl")
# 4: 26.66   6: 19.74   8: 15.10
# Confirma: mas cilindros = menos millas por galon.

Cuándo conviene factor ordinal: modelos lineales con factores ordinales usan contrastes polinómicos por defecto, en vez de comparar contra la categoría base. Para regresión exploratoria suele dar coeficientes más interpretables.

Ejercicio 13: explorar bikeshare con aggregate

  1. Calculá la media de bikers por estación con aggregate(bikers ~ season, data = bicis, mean). ¿Cuál es la estación con más uso?

  2. Calculá la media por día de la semana (weekday).

  3. Calculá la media por hora (hr). ¿En qué horas aparece el pico?

TipSolución 
# a)
aggregate(bikers ~ season, data = bicis, mean)
# invierno: 111,  primavera: 208,  verano: 236,  otono: 199
# El verano lidera (la gente sale mas en buen clima).

Ejercicio 14: cargar_y_describir

  1. Definí cargar_y_describir(archivo, sep = " ") que devuelva una lista con: df (el data.frame), dimensiones (vector de 2), tipos (vector con class por columna), n_na (vector con sum(is.na) por columna).

  2. Aplicala al CSV datos/bikeshare.csv. Pista: el archivo está separado por espacios, no por comas, así que pasá sep = " ".

  3. Mostrá los tipos y las columnas con más de cero NA.

TipSolución 
cargar_y_describir <- function(archivo, sep = " ") {
  df <- read.table(archivo, header = TRUE, sep = sep, stringsAsFactors = FALSE)
  list(
    df          = df,
    dimensiones = c(filas = nrow(df), columnas = ncol(df)),
    tipos       = sapply(df, class),
    n_na        = sapply(df, function(col) sum(is.na(col)))
  )
}

# b)
descripcion <- cargar_y_describir("datos/bikeshare.csv")
descripcion$dimensiones    # filas: 8645  columnas: 15

# c)
descripcion$tipos
# season int, mnth char, day int, ...
nas <- descripcion$n_na
nas[nas > 0]   # ninguna columna tiene NA

Cuidado al cargar: read.table con sep = " " colapsa múltiples espacios consecutivos como un único separador. Si el archivo usara separadores literales (un solo espacio entre cada campo), tendrías que usar read.delim o ajustar quote. Probá ?read.table para los detalles.

Ejercicio 15: is.numeric para identificar columnas

  1. Sobre el df de 14, identificá las columnas numéricas con sapply(df, is.numeric). Devolvé un vector de booleanos.

  2. Quedate solo con las columnas numéricas. Pista: df[, sapply(df, is.numeric)].

  3. Aplicá resumen_min (clase 1) a cada columna numérica con lapply. Guardá el resultado.

TipSolución 
df <- descripcion$df

# a)
es_num <- sapply(df, is.numeric)
es_num
# season, day, hr, weekday, workingday, temp, atemp, hum, windspeed,
# casual, registered, bikers: TRUE
# mnth, holiday, weathersit: FALSE

# b)
df_num <- df[, es_num]
ncol(df_num)   # 12

# c)
resumenes <- lapply(df_num, resumen_min)
length(resumenes)        # 12
resumenes$bikers         # media: 144,  min: 1,  max: 977

Por qué lapply y no sapply: resumen_min devuelve una lista (con 4 elementos). sapply intentaría simplificar a una matriz 4x12, que también funcionaría. lapply deja una lista de listas, que es más explícito y se accede con $. Cualquiera de las dos formas es válida; la elección es de estilo.

Ejercicio 16: la columna con mayor amplitud

  1. Para cada columna numérica, calculá max - min (la amplitud).

  2. Identificá la columna con mayor amplitud y la de menor amplitud.

  3. Justificá por qué bikers y temp tienen amplitudes muy distintas.

TipSolución 
# a)
amplitudes <- sapply(df_num, function(c) max(c) - min(c))
amplitudes

# b)
which.max(amplitudes)   # bikers (rango ~ 977)
which.min(amplitudes)   # workingday o holiday (rango = 1, son booleanos)

# c)
# bikers es un conteo: va de 1 a casi 1000.
# temp esta NORMALIZADO (0 a 1), por eso su amplitud es ~1.
# Comparar amplitudes brutas no tiene sentido si las unidades son distintas.
# Para comparar, hay que estandarizar: (x - mean) / sd, lo que hace `scale`.

Lección general: las amplitudes brutas no son comparables entre variables con escalas distintas. Es la principal razón para estandarizar antes de aplicar métodos sensibles a la escala (PCA, k-means, regularización).

Ejercicio 17: combinar listas y data.frame

  1. Construí una lista con dos data.frame: mtcars y iris[, 1:4].

  2. Aplicá nrow a cada uno con sapply.

  3. Aplicá colMeans a cada uno con lapply. ¿Por qué lapply y no sapply aquí?

TipSolución 
# a)
datos <- list(autos = mtcars, flores = iris[, 1:4])

# b)
sapply(datos, nrow)
# autos: 32   flores: 150

# c)
lapply(datos, colMeans)
# $autos: vector de 11 medias
# $flores: vector de 4 medias

Por qué no sapply: los colMeans de cada data.frame tienen distintas longitudes (11 en mtcars, 4 en iris[,1:4]). sapply no puede simplificar a matriz porque no son rectangulares. Termina devolviendo una lista (igual que lapply), así que conviene ser explícitos con lapply.

Ejercicio 18: combinar data.frame con do.call(rbind, ...)

  1. Construí tres data.frame chiquitos con la misma estructura (mismas columnas).

  2. Combinálos verticalmente con do.call(rbind, lista_de_dfs). Verificá dim.

  3. Hacé lo mismo con tres subsetes de airquality (uno por mes 5, 6, 7).

TipSolución 
# a)
df1 <- data.frame(x = 1:2, y = c("a", "b"))
df2 <- data.frame(x = 3:4, y = c("c", "d"))
df3 <- data.frame(x = 5:6, y = c("e", "f"))

# b)
combinado <- do.call(rbind, list(df1, df2, df3))
dim(combinado)   # 6 2

# c)
mayo  <- airquality[airquality$Month == 5, ]
junio <- airquality[airquality$Month == 6, ]
julio <- airquality[airquality$Month == 7, ]
verano <- do.call(rbind, list(mayo, junio, julio))
nrow(verano)   # 92

Por qué do.call: rbind(df1, df2, df3) funciona si tengo los dfs separados. Pero si están en una lista (variable largo), necesito do.call(rbind, lista), que despacha cada elemento como argumento. Es el patrón canónico para “aplicar rbind a una colección variable”.

Ejercicio 20: una función, una responsabilidad

  1. Mirá tu cargar_y_describir del ejercicio 14: ¿hace una sola cosa o varias? Listá las responsabilidades por separado.

  2. Partila en cuatro funciones: cargar_csv(archivo, sep), dimensiones_df(df), tipos_columnas(df), na_por_columna(df). Reescribí cargar_y_describir para que llame a las cuatro.

  3. ¿Qué función es más fácil de testear ahora? ¿Cuál se podría reusar en otro contexto?

TipSolución 
# a) Responsabilidades de cargar_y_describir:
#    1. abrir un archivo CSV
#    2. medir dimensiones
#    3. extraer tipos por columna
#    4. contar NAs por columna
#    5. ensamblar todo en una lista

# b)
cargar_csv <- function(archivo, sep = " ") {
  read.table(archivo, header = TRUE, sep = sep, stringsAsFactors = FALSE)
}

dimensiones_df <- function(df) {
  c(filas = nrow(df), columnas = ncol(df))
}

tipos_columnas <- function(df) sapply(df, class)

na_por_columna <- function(df) sapply(df, function(c) sum(is.na(c)))

cargar_y_describir <- function(archivo, sep = " ") {
  df <- cargar_csv(archivo, sep)
  list(
    df          = df,
    dimensiones = dimensiones_df(df),
    tipos       = tipos_columnas(df),
    n_na        = na_por_columna(df)
  )
}

# c)
# Cada funcion chica es testeable de forma aislada:
#   tipos_columnas(data.frame(a = 1, b = "x"))   # devuelve c("integer", "character")
# Y son reusables: tipos_columnas se puede aplicar a cualquier df,
# no solo al de bikeshare.

Beneficio en concreto: si mañana quiero describir un data.frame que ya tengo en memoria (no viene de CSV), uso dimensiones_df, tipos_columnas, na_por_columna directamente. Si todo estuviera dentro de cargar_y_describir, tendría que duplicar el código o pasarlo de fuerza por un archivo intermedio.
