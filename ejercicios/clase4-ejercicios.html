Ejercicio 1: histograma con varios breaks

  1. Hacé tres histogramas de airquality$Ozone con breaks = 10, breaks = 20, breaks = 50. Compará lo que se ve.

  2. En el histograma con breaks = 20, agregá título y eje en español.

  3. Marcá la media con abline(v = mean(airquality$Ozone, na.rm = TRUE), col = "red", lwd = 2).

TipSolución 
# a)
par(mfrow = c(1, 3))
hist(airquality$Ozone, breaks = 10,  main = "breaks = 10")
hist(airquality$Ozone, breaks = 20,  main = "breaks = 20")
hist(airquality$Ozone, breaks = 50,  main = "breaks = 50")
par(mfrow = c(1, 1))
# breaks=10 esconde la cola, breaks=50 introduce ruido,
# breaks=20 es un buen compromiso.

# b, c)
hist(airquality$Ozone,
     breaks = 20,
     main = "Ozono diario en NY (1973)",
     xlab = "Ozono (ppb)",
     col  = "lightgray", border = "white")
abline(v = mean(airquality$Ozone, na.rm = TRUE), col = "red", lwd = 2)

Curiosidad sobre breaks: R no necesariamente respeta el número exacto que pidas. breaks = 20 es una sugerencia: internamente busca puntos de corte “lindos” cerca de ese número. Para forzar un valor exacto, pasá un vector: breaks = seq(0, 200, by = 10).

Ejercicio 2: boxplot por grupo

  1. Boxplot de Temp ~ Month sobre airquality. Pintá las cajas con col = "lightblue".

  2. Convertí Month en factor con etiquetas c("May", "Jun", "Jul", "Ago", "Sep") y volvé a graficar. Pista: factor(airquality$Month, labels = c("May", ...)).

  3. ¿Qué mes tiene la mediana más alta? ¿Cuál tiene mayor dispersión (rango intercuartílico)?

TipSolución

        xlab = "Mes", ylab = "Temperatura (F)")

# b)

                         labels = c("May", "Jun", "Jul", "Ago", "Sep"))
boxplot(Temp ~ Mes, data = airquality, col = "lightblue",
        xlab = "Mes", ylab = "Temperatura (F)")

Curiosidad: los meses con mayor IQR son típicamente los de transición (mayo y septiembre): la temperatura varía mucho entre el inicio y el final del mes. En verano (junio-agosto) la dispersión es más chica porque el clima es más estable.

  3. Agregá una línea horizontal en la media de mpg con abline(h = mean(mtcars$mpg), lty = 2) y una leyenda con legend("topright", legend = c(...), col = c(...), lty = c(1, 2)).

TipSolución

Ejercicio 4: graficar_distribucion

  1. Definí graficar_distribucion(x, etiqueta = "") que dibuje histograma + boxplot lado a lado en par(mfrow = c(1, 2)). Restablecé el layout con par(mfrow = c(1, 1)) al final.

  2. Aplicala a airquality$Ozone y airquality$Wind.

  3. Aplicala a mtcars$mpg.

TipSolución 
graficar_distribucion <- function(x, etiqueta = "") {
  par(mfrow = c(1, 2))
  hist(x, main = paste("Histograma:", etiqueta),
       xlab = etiqueta, col = "gray85", border = "white")
  boxplot(x, main = paste("Boxplot:", etiqueta),
          horizontal = TRUE, col = "lightblue")
  par(mfrow = c(1, 1))
}

# b)
graficar_distribucion(airquality$Ozone, "Ozono (ppb)")
graficar_distribucion(airquality$Wind,  "Viento (mph)")

# c)
graficar_distribucion(mtcars$mpg, "Consumo (mpg)")

Mejor con on.exit: la versión actual restablece par solo si la función termina sin errores. Si hist o boxplot fallan, el layout queda en 1x2 para todos los gráficos siguientes. La solución correcta usa on.exit, lo vemos en el ejercicio 5.

Ejercicio 5: comparar con la versión auxiliar

  1. Cargá auxiliares.R y mirá la función graficar_hist_box provista.

  2. Comparala con tu graficar_distribucion. ¿Qué hace on.exit(par(par_anterior))? Pista: ?on.exit.

  3. Reescribí graficar_distribucion agregando on.exit para que el layout siempre se restaure, incluso si la función falla.

TipSolución 
# a, b)
# graficar_hist_box (en auxiliares.R) usa:
#   par_anterior <- par(mfrow = c(1, 2))
#   on.exit(par(par_anterior))
#
# on.exit registra una expresion que se ejecuta cuando la funcion sale,
# tanto si termina normal como si lanza error. Garantiza que el layout
# se restaura siempre.

# c)
graficar_distribucion <- function(x, etiqueta = "") {
  par_anterior <- par(mfrow = c(1, 2))
  on.exit(par(par_anterior))
  hist(x, main = paste("Histograma:", etiqueta),
       xlab = etiqueta, col = "gray85", border = "white")
  boxplot(x, main = paste("Boxplot:", etiqueta),
          horizontal = TRUE, col = "lightblue")
}

on.exit en general: sirve para limpiar recursos: cerrar archivos (close), liberar conexiones, restaurar opciones (options, par), borrar archivos temporales (unlink). Patrón canónico: la primera línea de la función abre el recurso, la segunda registra on.exit para cerrarlo. Es el equivalente al try/finally de Python.

Ejercicio 6: resumen por mes con barras

  3. Agregá barras de error usando la función auxiliar barplot_con_error(x, grupo, ...) (de auxiliares.R). Es un helper custom que calcula media y error estándar por grupo y los grafica de un solo viaje. Mirá su código si querés ver cómo se construye con aggregate + arrows.

Ejercicio 8: eda_minimo

  1. Definí eda_minimo(df) que: imprima str(df), summary(df), head(df, 3), y para cada columna numérica llame a graficar_distribucion.

  2. Aplicala a airquality.

  3. Aplicala a mtcars.

TipSolución 
eda_minimo <- function(df) {
  cat("Estructura:\n");  str(df)
  cat("\nResumen:\n");   print(summary(df))
  cat("\nPrimeras filas:\n"); print(head(df, 3))

  for (nombre in colnames(df)) {
    if (is.numeric(df[[nombre]])) {
      graficar_distribucion(df[[nombre]], etiqueta = nombre)
      readline("Enter para continuar...")   # opcional, para pausar
    }
  }
}

# b)
eda_minimo(airquality)

# c)
eda_minimo(mtcars)

Caso real: este patrón (“para cada columna, hacé X”) es la base de los reportes automáticos. Paquetes como skimr y DataExplorer están construidos sobre la misma idea pero con presentación más prolija. Si querés ver una versión “industrial”, probá skimr::skim(airquality).

Ejercicio 9: NAs en datos reales

  1. Identificá las columnas de airquality con NA. ¿Cuántos hay en cada una?

  2. Comparar el resultado de mean(airquality$Ozone) con mean(airquality$Ozone, na.rm = TRUE).

  3. Definí media_robusta(x) que devuelva mean(x, na.rm = TRUE) con un mensaje (message()) si encuentra NA. Probala.

TipSolución 
# a)
sapply(airquality, function(c) sum(is.na(c)))
# Ozone: 37, Solar.R: 7, otras: 0

# b)
mean(airquality$Ozone)                # NA
mean(airquality$Ozone, na.rm = TRUE)  # 42.13

# c)
media_robusta <- function(x) {
  n_na <- sum(is.na(x))
  if (n_na > 0) message("media_robusta: ignorando ", n_na, " NA")
  mean(x, na.rm = TRUE)
}
media_robusta(airquality$Ozone)
# message: ignorando 37 NA
# 42.13

Decisión silenciosa: mean(..., na.rm = TRUE) es práctico pero oculta información: no sabés cuántos faltantes había. media_robusta deja una huella audible. En análisis exploratorios, este tipo de función ayuda a detectar problemas de datos que de otra forma pasarían inadvertidos.

Ejercicio 10: normalizar con tests

  1. Definí normalizar(x) que devuelva (x - mean(x, na.rm = TRUE)) / sd(x, na.rm = TRUE).

  2. Validala con dos stopifnot sobre c(1, 2, 3, 4, 5): media cercana a 0 y desvío cercano a 1, ambas con tolerancia 1e-10.

  3. Aplicala a airquality$Wind. Graficá el histograma del resultado.

TipSolución 
normalizar <- function(x) {
  (x - mean(x, na.rm = TRUE)) / sd(x, na.rm = TRUE)
}

# b)
z <- normalizar(c(1, 2, 3, 4, 5))
stopifnot(abs(mean(z))     < 1e-10)
stopifnot(abs(sd(z) - 1)   < 1e-10)

# c)
hist(normalizar(airquality$Wind),
     main = "Wind normalizada (z-score)",
     xlab = "z", breaks = 20)

Convención del z-score: la función scale(x) de R hace lo mismo pero devuelve una matriz de una columna, no un vector. Por eso es común escribir nuestra propia versión que devuelva el mismo tipo que recibe.

Ejercicio 12: distribución de viajes por hora

  1. Histograma de bicis$bikers. ¿Por qué tiene una cola tan larga?

  2. Acotá el eje x al percentil 99: xlim = c(0, quantile(bicis$bikers, 0.99)). Volvé a graficar.

  3. Hacé el histograma de log10(bicis$bikers + 1). Compará con la versión sin transformar. ¿Por qué + 1?

TipSolución 
# a)
hist(bicis$bikers, breaks = 50)
# Cola larga: muchas horas tienen 1-50 viajes (madrugada, lluvia),
# pero algunas tienen 700+ (hora pico de buen tiempo).

# b)
hist(bicis$bikers,
     xlim = c(0, quantile(bicis$bikers, 0.99)),
     breaks = 50,
     main = "Bikers por hora (acotado al p99)")

# c)
hist(log10(bicis$bikers + 1),
     breaks = 50,
     main = "log10(bikers + 1)",
     xlab = "log10")

Por qué + 1: log10(0) es -Inf, lo que rompe el histograma. Sumar 1 (transformación log1p) convierte el cero en log10(1) = 0. Es el truco estándar para datos de conteos con ceros.

Forma resultante: la transformación log10 aplana la cola larga y la distribución se ve más simétrica. Eso facilita usar técnicas que asumen normalidad (regresión, t-test).

Ejercicio 13: viajes por estación

  1. Boxplot de bikers ~ season sobre bicis. ¿Qué estación tiene mayor mediana?

  2. Boxplot de bikers ~ weekday. ¿Hay diferencia entre días hábiles y fines de semana?

  3. Calculá la media de bikers por estación y graficá con barplot. Compará con el boxplot.

TipSolución 
# a)
boxplot(bikers ~ season, data = bicis, col = "lightblue")
# Verano y otoño tienen las medianas mas altas; invierno la mas baja.

# b)
boxplot(bikers ~ weekday, data = bicis, col = "lightblue")
# Las medianas son parecidas. La dispersion es ligeramente menor en
# fin de semana (sin pico tan alto de hora pico).

Ejercicio 14: el ciclo diario

  1. Calculá la media de bikers por hora con aggregate(bikers ~ hr, data = bicis, mean).

  2. Graficá la curva resultante con plot(... , type = "l", lwd = 2). Identificá los picos.

  3. Hacé lo mismo separando por workingday (días hábiles vs fines de semana). Pista: dos aggregate y dos lines sobre el mismo plot.

TipSolución 
# a, b)

plot(por_hora$hr, por_hora$bikers,
     type = "l", lwd = 2,
     xlab = "Hora", ylab = "Bikers medios",
     main = "Ciclo diario de uso")
# Picos a las 8h y 17h (ida y vuelta del trabajo).

Ejercicio 15: temperatura y uso

  1. Scatter de bikers ~ temp sobre bicis con pch = 16, col = rgb(0, 0, 0, 0.05). ¿Por qué alpha = 0.05?

  2. Agregá la recta de regresión abline(lm(bikers ~ temp, data = bicis), col = "red", lwd = 2).

  3. Calculá cor(bicis$bikers, bicis$temp). Interpretá el signo y la magnitud (entre -1 y 1).

TipSolución 
# a)
plot(bicis$temp, bicis$bikers,
     pch = 16, col = rgb(0, 0, 0, 0.05),
     xlab = "Temperatura (normalizada)",
     ylab = "Bikers",
     main = "Bikers vs temperatura")

Truco de los datasets grandes: con 10k+ puntos, pch = 16 y col = rgb(0,0,0, alpha) es la forma más simple de hacer un “density plot” en base R. Para un density plot real: smoothScatter(x, y) o MASS::kde2d.

Ejercicio 17: bootstrap con bikeshare

  1. Aplicá bootstrap_media (clase 2) a bicis$bikers con 1000 réplicas.

  2. Calculá un IC del 95% por percentiles.

  3. Graficá el histograma de las medias bootstrap usando la función auxiliar graficar_bootstrap (de auxiliares.R). Mirá su código antes de usarla.

TipSolución 
# a)
set.seed(42)
boot <- bootstrap_media(bicis$bikers, n_rep = 1000)
length(boot)   # 1000

# b)
quantile(boot, c(0.025, 0.975))
# ~ (140.5, 147.5)

Por qué el IC es tan angosto: cuando n es grande (8645), el error estándar de la media (sd(x) / sqrt(n)) es chico. El bootstrap también tiene baja dispersión. Para muestras chicas (n < 30), el IC es más ancho y el bootstrap aporta más.

Ejercicio 18: regresión lineal múltiple

  1. Ajustá mod <- lm(bikers ~ temp + hum + windspeed, data = bicis). Mostrá summary(mod). Lo importante en este momento es leer la salida y entender qué reporta.

  2. Identificá los coeficientes estimados y su signo. ¿Cuál variable parece tener más peso?

  3. Calculá confint(mod) y coef(mod). Compará intervalos con coeficientes.

TipSolución 
mod <- lm(bikers ~ temp + hum + windspeed, data = bicis)
summary(mod)
# Estimate, Std. Error, t value, Pr(>|t|) por cada coeficiente.
# Mas un R^2 al final.

# b)
coef(mod)
# (Intercept), temp, hum, windspeed
# temp: positivo (mas calor, mas viajes)
# hum: negativo (mas humedad, menos viajes)

Cuidado al interpretar: los coeficientes están en la escala original de cada variable. Como temp, hum y windspeed están normalizados (0 a 1), sus magnitudes SÍ son comparables acá. Pero si una estuviera en °C y otra en mph, no podríamos compararlos directamente sin estandarizar.

eda_minimo <- function(df) {
  cat("Estructura:\n");  str(df)
  cat("\nResumen:\n");   print(summary(df))
  cat("\nPrimeras filas:\n"); print(head(df, 3))

  cat("\nNAs por columna:\n")
  print(sapply(df, function(c) sum(is.na(c))))

  cat("\nFactores:\n")
  for (nombre in colnames(df)) {
    if (is.factor(df[[nombre]])) {
      cat("\n  ", nombre, ":\n")
      print(table(df[[nombre]]))
    }
  }

  for (nombre in colnames(df)) {
    if (is.numeric(df[[nombre]])) {
      graficar_distribucion(df[[nombre]], etiqueta = nombre)
    }
  }
}

TipSolución 
# a)
normalizar(5)         # NaN  (sd de un solo valor es NA)

# b)
normalizar <- function(x) {
  stopifnot(length(x) >= 2,
            sd(x, na.rm = TRUE) > 0)
  (x - mean(x, na.rm = TRUE)) / sd(x, na.rm = TRUE)
}

# c)
tryCatch(
  normalizar(c(1, 1, 1)),
  error = function(e) message("Falla esperada: ", conditionMessage(e))
)
# Falla esperada: sd(x, na.rm = TRUE) > 0 is not TRUE

Filosofía de stopifnot: las precondiciones explícitas son documentación ejecutable. Cualquiera que lea stopifnot(length(x) >= 2) sabe inmediatamente que la función espera al menos dos elementos. Sin la línea, el error sería Error in (x - mean(x))/sd(x): NaN/numeric value..., mucho más oscuro.
