Clase 2 - Ejercicios

Condicionales y bucles aplicados a airquality

Contexto

Vamos a usar airquality, mediciones diarias de calidad del aire en Nueva York entre mayo y septiembre de 1973. Tiene 153 filas, 6 columnas y valores faltantes (NA) reales.

head(airquality)
?airquality

A las funciones de la clase 1 (etiquetar_auto, dif_promedio, dentro_rango, resumen_min) sumamos clasificar_temp, contar_eventos, primer_indice_que_cumple, bootstrap_media y clasificar_presion.

Algunos ejercicios usan helpers de auxiliares.R (cargalo con source("ejercicios/auxiliares.R")). Esos helpers son piezas custom que usamos como caja negra para no extender el ejercicio: por ejemplo, benchmark_funcion. Para entenderlos podés leer su código en el archivo.

Ejercicio 2: if simple

  1. Asigná temp_observada <- 85 (en °F). Escribí un bloque if que imprima "alerta de calor" si temp_observada >= 80.

  2. Reasigná temp_observada <- 70 y volvé a correr. ¿Imprime algo?

  3. ¿Qué pasa si temp_observada es NA? Probá. Identificá el problema.

TipSolución 
# a)
temp_observada <- 85
if (temp_observada >= 80) print("alerta de calor")

# b)
temp_observada <- 70
if (temp_observada >= 80) print("alerta de calor")
# no imprime nada (la condicion es FALSE)

# c)
temp_observada <- NA
if (temp_observada >= 80) print("alerta de calor")
# Error: missing value where TRUE/FALSE needed

Por qué falla con NA: la comparación NA >= 80 devuelve NA (no TRUE ni FALSE), e if requiere un valor lógico definido. Hay que decidir explícitamente qué hacer con NA. Patrón común: if (!is.na(x) && x >= 80).

Ejercicio 4: if / else if / else

  1. Definí clasificar_temp(temp) (en °F): "frio" si temp < 60, "templado" si 60 <= temp < 80, "caluroso" si temp >= 80.

  2. Probala con c(50, 70, 90, 100) dentro de un for.

  3. Hacé lo mismo con ifelse anidado vectorial. Compará legibilidad y velocidad (system.time).

TipSolución 
clasificar_temp <- function(temp) {
  if (is.na(temp))       NA
  else if (temp < 60)    "frio"
  else if (temp < 80)    "templado"
  else                   "caluroso"
}

# b)
for (t in c(50, 70, 90, 100)) {
  print(clasificar_temp(t))
}
# "frio"  "templado"  "caluroso"  "caluroso"

# c) Version vectorizada con ifelse anidado
clasificar_temp_vec <- function(temp) {
  ifelse(is.na(temp), NA,
  ifelse(temp < 60,   "frio",
  ifelse(temp < 80,   "templado",
                      "caluroso")))
}

system.time(replicate(1000, sapply(airquality$Temp, clasificar_temp)))
system.time(replicate(1000, clasificar_temp_vec(airquality$Temp)))
# La version vectorizada es ~50x mas rapida.

Trade-off: la versión con if/else if/else se lee más natural (“cae en este caso, sino en este otro…”). ifelse anidado es más rápido y vectorizado pero menos legible cuando hay muchos casos. Cuando son más de 3-4 niveles, cut(temp, breaks = c(-Inf, 60, 80, Inf), labels = c("frio", "templado", "caluroso")) es la forma más prolija.

Ejercicio 5: operadores lógicos en condiciones

  1. Definí dia_alerta(temp, ozono) que devuelva TRUE si ambos superan sus umbrales: temp >= 85 Y ozono >= 80.

  2. Probala con dia_alerta(90, 85) y dia_alerta(70, 90).

  3. Aplicala con un for a las primeras 30 filas de airquality. Cuidado con NA: tratalos como FALSE.

TipSolución 
dia_alerta <- function(temp, ozono) {
  if (is.na(temp) || is.na(ozono)) return(FALSE)
  temp >= 85 && ozono >= 80
}

# b)
dia_alerta(90, 85)   # TRUE
dia_alerta(70, 90)   # FALSE (temp baja)

# c)
n_alertas <- 0
for (i in 1:30) {
  if (dia_alerta(airquality$Temp[i], airquality$Ozone[i])) {
    n_alertas <- n_alertas + 1
  }
}
n_alertas

Detalle: dentro de if usamos && y || (corto-circuito, escalares). Para vectores usaríamos & y |. Si te confunden, && es para condiciones de un solo valor; & es para vectores enteros.

Ejercicio 7: for con seq_along

  1. Recorré airquality$Ozone con seq_along. ¿Por qué seq_along es más seguro que 1:length(...)? Pista: pensá qué pasa si length es 0.

  2. Calculá la suma de los valores no-NA de Ozone con un for. Cuidado con los NA: salteálos con next.

  3. Compará con sum(airquality$Ozone, na.rm = TRUE).

TipSolución 
# a)
# Si x tiene length 0:
x <- numeric(0)
length(x)        # 0
1:length(x)      # c(1, 0)  <-- bucle ejecuta DOS veces, indices invalidos
seq_along(x)     # integer(0)  <-- bucle no ejecuta, correcto

# b)
suma <- 0
for (i in seq_along(airquality$Ozone)) {
  if (is.na(airquality$Ozone[i])) next
  suma <- suma + airquality$Ozone[i]
}
suma   # 4887

# c)
sum(airquality$Ozone, na.rm = TRUE)   # 4887

Curiosidad: el bug de 1:length(x) con x vacío es uno de los más clásicos en R. Pasa silenciosamente porque 1:0 es c(1, 0), el bucle entra dos veces, e índices x[1] y x[0] devuelven NA o vacío. La salida puede parecer correcta pero está mal.

Ejercicio 8: acumular en un vector pre-asignado

  1. Calculá Ozone^2 - Wind para cada día con un for. Pre-asigná un vector numeric(nrow(airquality)) y rellenalo. Cuidado con NA (dejalos como NA).

  2. Hacé lo mismo en una línea vectorizada: airquality$Ozone^2 - airquality$Wind. Verificá que coincide.

  3. Compará el tiempo de las dos versiones con la función auxiliar benchmark_funcion(fn, n_rep = 10) (de auxiliares.R). Reportá la media y el desvío. ¿Cuántas veces más rápida es la versión vectorizada?

TipSolución 
source("ejercicios/auxiliares.R")

# a)
calc_for <- function() {
  out <- numeric(nrow(airquality))
  for (i in seq_len(nrow(airquality))) {
    out[i] <- airquality$Ozone[i]^2 - airquality$Wind[i]
  }
  out
}

# b)
calc_vec <- function() {
  airquality$Ozone^2 - airquality$Wind
}

identical(calc_for(), calc_vec())   # TRUE

# c)
benchmark_funcion(calc_for, n_rep = 100)
# media ~0.0008s, sd ~0.0001s
benchmark_funcion(calc_vec, n_rep = 100)
# media ~0.00002s, sd ~0.000003s
# La version vectorizada es ~30-50x mas rapida en este dataset.

Por qué tanta diferencia: las operaciones vectorizadas en R están implementadas en C, mientras que el for interpreta cada iteración. La brecha crece con el tamaño del vector. Con airquality (153 filas) ya se nota; con un millón de filas la diferencia es decisiva.

Ejercicio 9: for anidado

  1. Construí una matriz M de 5x5 donde M[i, j] = i * j con dos for anidados.

  2. Verificá con outer(1:5, 1:5, "*"). Coinciden.

  3. Calculá la suma de los elementos pares de M recorriéndola con dos for anidados.

TipSolución 
# a)
M <- matrix(0, nrow = 5, ncol = 5)
for (i in 1:5) {
  for (j in 1:5) {
    M[i, j] <- i * j
  }
}

# b)
identical(M, outer(1:5, 1:5, "*"))   # TRUE

# c)
suma_pares <- 0
for (i in 1:5) {
  for (j in 1:5) {
    if (M[i, j] %% 2 == 0) suma_pares <- suma_pares + M[i, j]
  }
}
suma_pares   # 90

# vectorizado:
sum(M[M %% 2 == 0])   # 90

Tabla que sale gratis: M construida con i * j es la tabla de multiplicar del 1 al 5. outer generaliza esto a cualquier función binaria: outer(1:3, 1:3, "+") da la tabla de sumar.

Ejercicio 10: while básico

  1. Escribí un while que duplique x = 1 mientras x < 1000. ¿En qué valor se detiene?

  2. Modificalo para que cuente cuántas iteraciones hizo. Imprimí el contador final.

  3. Compará con ceiling(log2(1000)).

TipSolución 
# a)
x <- 1
while (x < 1000) x <- x * 2
x   # 1024

# b)
x <- 1
iter <- 0
while (x < 1000) {
  x <- x * 2
  iter <- iter + 1
}
iter   # 10

# c)
ceiling(log2(1000))   # 10

Por qué coincide: después de n duplicaciones, x = 2^n. Buscamos el primer n tal que 2^n >= 1000, que es ceiling(log2(1000)) = 10. Esto generaliza: si arrancás con x y duplicás hasta superar M, hacés ceiling(log2(M / x)) iteraciones.

Ejercicio 11: while con tolerancia

  1. Escribí un while que estime sqrt(2) con la iteración Babilónica: x <- (x + 2/x) / 2, partiendo de x = 1. Detenete cuando abs(x^2 - 2) < 1e-10.

  2. Imprimí el resultado y la cantidad de iteraciones.

  3. Compará con sqrt(2).

TipSolución 
# a, b)
x <- 1
iter <- 0
while (abs(x^2 - 2) > 1e-10) {
  x <- (x + 2/x) / 2
  iter <- iter + 1
}
x      # 1.414214
iter   # 5

# c)
sqrt(2) - x   # ~ 1.6e-12  (mucho mas chico que 1e-10)

Curiosidad: la iteración Babilónica es un caso particular de Newton-Raphson aplicado a f(x) = x^2 - 2. Converge cuadráticamente: la cantidad de dígitos correctos se duplica en cada paso. Con 5 iteraciones ya tenés ~12 dígitos correctos.

Ejercicio 12: break para detenerse al encontrar

  1. Recorré airquality$Ozone y detenete con break apenas encontrés un NA. Imprimí en qué índice se detuvo.

  2. Verificá con which(is.na(airquality$Ozone))[1].

  3. Modificalo para detenerte al primer valor > 100. Compará con which(airquality$Ozone > 100)[1].

TipSolución 
# a)
for (i in seq_along(airquality$Ozone)) {

  }
}
# 5

Ejercicio 13: next para saltear

  1. Recorré los primeros 30 días de airquality$Ozone. Salteá los NA con next e imprimí solo los valores numéricos.

  2. Imprimí solo los valores > 80, salteando NA con next.

  3. Imprimí solo los días impares con valores > 50. Combiná dos next.

TipSolución 
# a)
for (i in 1:30) {
  if (is.na(airquality$Ozone[i])) next
  print(airquality$Ozone[i])
}

# b)
for (i in 1:30) {
  v <- airquality$Ozone[i]
  if (is.na(v)) next
  if (v <= 80)  next
  print(v)
}

# c)
for (i in 1:30) {
  v <- airquality$Ozone[i]
  if (i %% 2 == 0) next
  if (is.na(v) || v <= 50) next
  cat("dia", i, ":", v, "\n")
}

Estilo: múltiples next al inicio del cuerpo del bucle son una forma común de “guard clauses”. Mantienen el cuerpo principal con poca indentación. Es preferible a anidar varios if.

Ejercicio 14: contar_eventos

  1. Definí contar_eventos(x, umbral) que use un for para contar cuántos elementos de x son > umbral. Cuidado con NA.

  2. Probala con airquality$Ozone y umbral = 80 (alerta de la OMS).

  3. Generalizala a contar_eventos(x, umbral, comparacion = ">") aceptando ">", "<", ">=", "<=". Pista: match.arg. Probala con comparacion = "<".

TipSolución 
# a)
contar_eventos <- function(x, umbral) {
  n <- 0
  for (v in x) {
    if (!is.na(v) && v > umbral) n <- n + 1
  }
  n
}

# b)
contar_eventos(airquality$Ozone, 80)   # 17

# c)
contar_eventos <- function(x, umbral, comparacion = c(">", "<", ">=", "<=")) {
  comparacion <- match.arg(comparacion)
  cmp <- match.fun(comparacion)
  n <- 0
  for (v in x) {
    if (!is.na(v) && cmp(v, umbral)) n <- n + 1
  }
  n
}

contar_eventos(airquality$Ozone, 20, "<")   # ~30

match.arg y match.fun: dos de las herramientas más útiles en funciones genéricas. match.arg valida que el argumento sea uno de los permitidos. match.fun resuelve un string al objeto función correspondiente. Combinadas, dan funciones flexibles y robustas.

Ejercicio 15: primer_indice_que_cumple

  1. Definí primer_indice_que_cumple(x, condicion_fun) con un while y seq_along. condicion_fun es una función que devuelve TRUE/FALSE por elemento. Devolvé NA si no se cumple para ninguno.

  2. Probala con condicion_fun = function(z) !is.na(z) && z > 100 sobre airquality$Ozone.

  3. Compará con which(airquality$Ozone > 100 & !is.na(airquality$Ozone))[1].

TipSolución 
primer_indice_que_cumple <- function(x, condicion_fun) {
  i <- 1
  while (i <= length(x)) {
    if (condicion_fun(x[i])) return(i)
    i <- i + 1
  }
  NA
}

# b)
primer_indice_que_cumple(
  airquality$Ozone,
  function(z) !is.na(z) && z > 100
)
# 30

# c)
which(airquality$Ozone > 100 & !is.na(airquality$Ozone))[1]   # 30

Diferencia conceptual: la versión while recorre hasta encontrar y corta. which evalúa la condición sobre TODO el vector y devuelve el primer índice. Para vectores chicos no importa; para uno de millones de elementos, which puede gastar memoria y tiempo.

Ejercicio 16: clasificar_presion

  1. Definí clasificar_presion(sistolica) según la clasificación AHA: < 90 "hipotenso", 90-119 "normal", 120-129 "elevada", 130-139 "hipertension 1", >= 140 "hipertension 2". Devolvé NA si la entrada es NA.

  2. Probala con c(85, 110, 125, 135, 150, NA).

  3. Aplicala con sapply a un vector de 100 valores aleatorios runif(100, 70, 180). Resumí con table.

TipSolución 
clasificar_presion <- function(sistolica) {
  if (is.na(sistolica))      NA
  else if (sistolica < 90)   "hipotenso"
  else if (sistolica < 120)  "normal"
  else if (sistolica < 130)  "elevada"
  else if (sistolica < 140)  "hipertension 1"
  else                       "hipertension 2"
}

# b)
sapply(c(85, 110, 125, 135, 150, NA), clasificar_presion)
# "hipotenso"  "normal"  "elevada"  "hipertension 1"  "hipertension 2"  NA

# c)
set.seed(42)
sis <- runif(100, 70, 180)
table(sapply(sis, clasificar_presion), useNA = "always")
# elevada  hipertension 1  hipertension 2  hipotenso  normal  <NA>

Curiosidad de runif: genera valores uniformes, no presiones reales. La distribución de presiones reales es aproximadamente normal centrada en ~120, no uniforme. El ejercicio enfatiza la función, no la modelación.

Ejercicio 17: bootstrap_media

  1. Definí bootstrap_media(x, n_rep = 1000) con replicate y sample. Devolvé el vector de medias bootstrap.

  2. Aplicala a airquality$Wind (no tiene NA). La media de las medias debería estar cerca de mean(airquality$Wind).

  3. Calculá un intervalo del 95% por percentiles: quantile(boot, c(0.025, 0.975)). Compará con t.test(airquality$Wind)$conf.int.

TipSolución 
bootstrap_media <- function(x, n_rep = 1000) {
  replicate(n_rep, mean(sample(x, replace = TRUE)))
}

# b)
set.seed(42)
boot <- bootstrap_media(airquality$Wind, 1000)
mean(boot)                  # ~9.96
mean(airquality$Wind)       # 9.96 (coincide en general, hay ruido de muestra)

# c)
quantile(boot, c(0.025, 0.975))   # ~ (9.40, 10.50)
t.test(airquality$Wind)$conf.int  # (9.39, 10.51)
# Casi identicos para una distribucion aproximadamente simetrica.

Para distribuciones asimétricas: el IC bootstrap suele ser mejor que el t-test, porque no asume normalidad. Conviene usarlo cuando los datos están sesgados (como Ozone).

Ejercicio 18: bootstrap de una proporción

  1. Definí bootstrap_proporcion(x, p, n_rep = 1000) que estime, vía bootstrap, la proporción de elementos de x que cumplen una condición lógica p. p es una función que devuelve TRUE/FALSE.

  2. Probala con airquality$Ozone y p = function(z) !is.na(z) && z > 80.

  3. Devolvé un IC del 95% para la proporción y comparalo con la proporción puntual.

TipSolución 
bootstrap_proporcion <- function(x, p, n_rep = 1000) {
  replicate(n_rep, {
    re_muestra <- sample(x, replace = TRUE)
    mean(sapply(re_muestra, p))
  })
}

# b)
set.seed(42)
boot_p <- bootstrap_proporcion(
  airquality$Ozone,
  function(z) !is.na(z) && z > 80,
  n_rep = 1000
)

# c)
prop_puntual <- mean(sapply(airquality$Ozone,
                            function(z) !is.na(z) && z > 80))
prop_puntual                       # 0.111
quantile(boot_p, c(0.025, 0.975))  # ~ (0.07, 0.16)

Mejora de eficiencia: dentro del replicate, evitar sapply y vectorizar la condición:

mask <- !is.na(airquality$Ozone) & airquality$Ozone > 80
boot_p <- replicate(1000, mean(sample(mask, replace = TRUE)))

Hace lo mismo, mucho más rápido.

Ejercicio 19: clasificar y resumir

  1. Aplicá clasificar_temp (ejercicio 4) a airquality$Temp y guardá el resultado en una columna nueva del data.frame.

  2. Resumí con table(airquality$temp_categoria). ¿Qué proporción de días son calurosos?

  3. Calculá la media de Ozone por categoría con aggregate(Ozone ~ temp_categoria, data = airquality, mean, na.rm = TRUE). Interpretá.

TipSolución 
# a)
airquality$temp_categoria <- sapply(airquality$Temp, clasificar_temp)

# b)
table(airquality$temp_categoria)
# caluroso: 32   frio: 9   templado: 112
prop.table(table(airquality$temp_categoria))
# caluroso ~21%, frio ~6%, templado ~73%

# c)
aggregate(Ozone ~ temp_categoria, data = airquality, mean, na.rm = TRUE)
# caluroso: ~73   frio: ~18   templado: ~32
# Mayor temperatura, mayor ozono. Tiene sentido fisicoquimico:
# las reacciones fotoquimicas que producen ozono son mas rapidas con calor.

Por qué tiene sentido: el ozono troposférico se forma por reacciones entre óxidos de nitrógeno y compuestos orgánicos volátiles, catalizadas por la radiación solar y la temperatura. Por eso los picos de ozono ocurren en días calurosos y soleados.

Ejercicio 20: comentar el porqué

  2. Mirá tu código del ejercicio 17: borrá los comentarios que solo parafrasean lo que hace cada línea.
