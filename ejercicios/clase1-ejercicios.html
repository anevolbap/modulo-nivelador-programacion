Clase 1 - Ejercicios

Tipos de datos, vectores y primeras funciones

Contexto

Vamos a usar mtcars, un dataset de 32 autos con 11 variables (consumo, cilindros, potencia, peso, etc.). Viene con R, no requiere instalación.

head(mtcars)
?mtcars

A lo largo de esta guía construimos una pequeña caja de herramientas que vamos a reusar en las clases siguientes: etiquetar_auto, dif_promedio, dentro_rango, resumen_min.

Las soluciones están en bloques colapsables debajo de cada ejercicio. Conviene probar primero y mirar la solución solo después.

Ejercicio 1: variables y tipos atómicos

  1. Asigná cuatro variables que describan el primer auto de mtcars: nombre_auto (string con rownames(mtcars)[1]), consumo_mpg (double, mtcars$mpg[1]), cilindros (integer, as.integer(mtcars$cyl[1])) y transmision_manual (logical, mtcars$am[1] == 1). Verificá class() de cada una.

  2. Verificá también typeof(). ¿En qué se diferencian class y typeof para consumo_mpg?

  3. ¿Qué devuelve is.numeric(cilindros)? ¿Y is.numeric(transmision_manual)? Justificá.

TipSolución 
# a)
nombre_auto        <- rownames(mtcars)[1]            # "Mazda RX4"
consumo_mpg        <- mtcars$mpg[1]                  # 21
cilindros          <- as.integer(mtcars$cyl[1])      # 6L
transmision_manual <- mtcars$am[1] == 1              # TRUE

class(nombre_auto)         # "character"
class(consumo_mpg)         # "numeric"
class(cilindros)           # "integer"
class(transmision_manual)  # "logical"

# b)
typeof(consumo_mpg)        # "double"  (class dice "numeric", typeof dice "double")
typeof(cilindros)          # "integer"

# c)
is.numeric(cilindros)           # TRUE   integer es subconjunto de numeric
is.numeric(transmision_manual)  # FALSE  logical no entra

Curiosidad: is.numeric también devuelve TRUE para integer. Si necesitás distinguir el tipo exacto, usá is.double o is.integer. En R, numeric es la clase paraguas para “número” y integer se considera un caso particular.

Ejercicio 2: coerción de tipos

  1. Predecí (sin correr) qué devuelve c(TRUE, 1L, 1.5). Después corrélo y compará con tu predicción.

  2. Predecí qué devuelve c(TRUE, 1, "a"). Justificá usando la regla de coerción.

  3. ¿Qué pasa al evaluar consumo_mpg + transmision_manual? Identificá qué tipo predomina.

TipSolución 
# a)
c(TRUE, 1L, 1.5)        # 1.0 1.0 1.5  (double — promueve TRUE a 1, integer a double)

# b)
c(TRUE, 1, "a")         # "TRUE" "1" "a"  (character — gana el más general)

# c)
consumo_mpg + transmision_manual    # 22  (TRUE coerce a 1; resultado numeric)

Regla: cuando se mezclan tipos en un vector o una operación, R promueve todo al tipo más general en la jerarquía logical → integer → double → character.

Caso no obvio: c(TRUE, NA) devuelve c(TRUE, NA) y class es logical. Pero c(1L, NA) da integer y c(1.5, NA) da double. NA se adapta al contexto.

Ejercicio 3: vector de consumos

  1. Asigná consumos <- mtcars$mpg. Verificá length(consumos) y class(consumos).

  2. Calculá mean, median, min y max de consumos. Compará con la salida de summary(consumos).

  3. Quedate con los consumos por encima del promedio usando indexado lógico. ¿Cuántos son?

TipSolución 
# a)
consumos <- mtcars$mpg
length(consumos)   # 32
class(consumos)    # "numeric"

# b)
mean(consumos)     # 20.09
median(consumos)   # 19.2
min(consumos)      # 10.4
max(consumos)      # 33.9
summary(consumos)  # min, Q1, mediana, media, Q3, max

consumos_altos <- consumos[consumos > mean(consumos)]

sum(consumos > mean(consumos))    # mismo resultado, mas idiomatico

Nota: mediana < media indica una distribución sesgada hacia valores grandes. Acá son cercanas (~19 vs ~20), así que el sesgo es leve.

Ejercicio 5: filtros con condiciones múltiples

  1. Filtrá los autos con mpg > 25. ¿Cuántos son?

  2. Filtrá los autos con mpg > 25 Y cyl == 4. Pista: operador &.

  3. Filtrá los autos con cyl == 4 o cyl == 8. Pista: | o %in%.

TipSolución 
# a)
sum(mtcars$mpg > 25)    # 6

# b)
sum(mtcars$mpg > 25 & mtcars$cyl == 4)   # 6
# los seis autos con mpg > 25 son todos de 4 cilindros

# c)
sum(mtcars$cyl %in% c(4, 8))   # 25 (todos menos los de 6 cilindros)
sum(mtcars$cyl == 4 | mtcars$cyl == 8)   # equivalente

Curiosidad: %in% es más legible que la cadena de | cuando hay tres o más valores. Internamente usa match. Para conjuntos grandes (>1000 valores) hay diferencias de performance, pero a esta escala dan lo mismo.

Ejercicio 6: función etiquetar_auto

  1. Definí etiquetar_auto(modelo, mpg) que devuelva el string "<modelo>: <mpg> mpg". Pista: paste0. Probala con "Mazda RX4" y 21.

  2. Probala con valores extraídos de mtcars: etiquetar_auto(rownames(mtcars)[1], mtcars$mpg[1]).

  3. Agregale un valor por defecto mpg = NA y observá la salida cuando se omite el segundo argumento.

TipSolución 
etiquetar_auto <- function(modelo, mpg = NA) {
  paste0(modelo, ": ", mpg, " mpg")
}

# a)
etiquetar_auto("Mazda RX4", 21)              # "Mazda RX4: 21 mpg"

# b)
etiquetar_auto(rownames(mtcars)[1], mtcars$mpg[1])   # "Mazda RX4: 21 mpg"

# c)
etiquetar_auto("Auto X")    # "Auto X: NA mpg"

Caso interesante: paste0 es vectorizado: etiquetar_auto(rownames(mtcars), mtcars$mpg) devuelve un vector con las 32 etiquetas en una sola llamada.

Ejercicio 7: función dif_promedio

  1. Definí dif_promedio(x) que devuelva x - mean(x). Aplicala a mtcars$mpg. Calculá la suma de las diferencias. Justificá el resultado teóricamente.

  2. ¿Cuántos autos están por encima del promedio? Pista: sum(dif_promedio(x) > 0).

  3. ¿Cuál es el auto que más se aleja del promedio en valor absoluto? Pista: which.max(abs(...)).

TipSolución 
dif_promedio <- function(x) x - mean(x)

# a)
sum(dif_promedio(mtcars$mpg))    # ~ 0  (en realidad un numero como 1.5e-14)
# Teorico: sum(x_i - mean(x)) = sum(x_i) - n*mean(x) = sum(x_i) - sum(x_i) = 0
# El residuo es ruido de punto flotante.

# b)
sum(dif_promedio(mtcars$mpg) > 0)   # 18

# c)
i <- which.max(abs(dif_promedio(mtcars$mpg)))
rownames(mtcars)[i]    # "Toyota Corolla" (el de mayor mpg)

Nota numérica: la suma “exacta” es 0 pero R devuelve algo como 1.5e-14 por aritmética de punto flotante. Comparar con == falla; usá abs(suma) < 1e-10.

Ejercicio 8: función dentro_rango

  1. Definí dentro_rango(x, a, b) que devuelva TRUE si a <= x <= b, FALSE en otro caso. La función debe ser vectorizada: si x es un vector, devuelve un vector de booleanos. Pista: &.

  2. Probala con dentro_rango(c(1, 5, 10, 15, 20), 5, 15).

  3. Aplicala a mtcars$mpg con rango c(20, 25). ¿Cuántos autos caen dentro del rango?

TipSolución 
dentro_rango <- function(x, a, b) x >= a & x <= b

# b)
dentro_rango(c(1, 5, 10, 15, 20), 5, 15)
# FALSE TRUE TRUE TRUE FALSE

# c)
sum(dentro_rango(mtcars$mpg, 20, 25))   # 8

Sutileza: la versión vectorizada usa & (operador elemento a elemento). && solo evalúa el primer elemento (cortocircuito) y se usa típicamente en condiciones de if. Confundirlos es un error común.

Ejercicio 9: conversión de unidades

  1. Definí mpg_a_km_l(x) que convierta de mpg (millas por galón) a km/L. Factor: 1 mpg ≈ 0.4251 km/L.

  2. Aplicala a mtcars$mpg para obtener un vector de consumos en km/L.

  3. Definí km_l_a_mpg(x) (función inversa). Verificá que km_l_a_mpg(mpg_a_km_l(20)) devuelve aproximadamente 20.

TipSolución 
mpg_a_km_l <- function(x) x * 0.4251
km_l_a_mpg <- function(x) x / 0.4251

# b)
mpg_a_km_l(mtcars$mpg)   # vector con 32 consumos en km/L

# c)
km_l_a_mpg(mpg_a_km_l(20))   # 20.0  (igual hasta el redondeo)

Curiosidad: el factor exacto es 1.609344 / 3.785411784 = 0.42514.... Si querés precisión, definí la constante con más decimales o leela de un paquete como units.

Ejercicio 10: función resumen_min

  1. Definí resumen_min(x) que reciba un vector numérico y devuelva una lista con media, mediana, min, max.

  2. Aplicala a mtcars$mpg. Verificá class() del resultado.

  3. Aplicala a mtcars$hp. Compará el rango con el de mpg.

TipSolución 
resumen_min <- function(x) {
  list(media   = mean(x),
       mediana = median(x),
       min     = min(x),
       max     = max(x))
}

# b)
resumen_min(mtcars$mpg)
# $media: 20.09  $mediana: 19.2  $min: 10.4  $max: 33.9
class(resumen_min(mtcars$mpg))   # "list"

# c)
resumen_min(mtcars$hp)
# rango: 52 a 335 (mucho mas amplio que mpg)

Caso no contemplado: si x tiene NA, todas las funciones devuelven NA. Para hacerla robusta agregaríamos un argumento na.rm = FALSE y lo pasaríamos a cada llamada. Lo vamos a ver en la guía 4.

Ejercicio 11: lista que describe un auto

  1. Creá una lista auto_1 con: modelo (string), mpg (double), cilindros (integer), caja ("manual" o "automatica"). Usá los datos del primer auto de mtcars.

  2. Accedé a cada campo con $, con [[ ]] y con [ ]. Mostrá class() de cada acceso.

  3. Cambiá el valor de mpg reasignándolo (auto_1$mpg <- ...). Verificá.

TipSolución 
auto_1 <- list(
  modelo     = rownames(mtcars)[1],
  mpg        = mtcars$mpg[1],
  cilindros  = as.integer(mtcars$cyl[1]),
  caja       = ifelse(mtcars$am[1] == 1, "manual", "automatica")
)

# b)
auto_1$modelo            # "Mazda RX4" (character)
auto_1[["modelo"]]       # "Mazda RX4" (character)
auto_1["modelo"]         # lista de 1 elemento (list)

class(auto_1$modelo)         # "character"
class(auto_1[["modelo"]])    # "character"
class(auto_1["modelo"])      # "list"

# c)
auto_1$mpg <- 22
auto_1$mpg   # 22

Atajo mental para listas: [[ ]] saca el contenido (un elemento), [ ] saca un sub-cajón (otra lista, posiblemente con varios elementos). $ es la versión “atajo” de [[ ]] con nombre.

Ejercicio 12: comparar dos autos

  1. Definí comparar_autos(auto_a, auto_b) que reciba dos listas con campos mpg y cilindros y devuelva una lista con dif_mpg y dif_cilindros.

  2. Probala con dos autos cualesquiera de mtcars.

  3. Si llamás comparar_autos(list(mpg = 20), list(consumo = 25)), ¿qué pasa? Identificá el problema.

TipSolución 
comparar_autos <- function(auto_a, auto_b) {
  list(
    dif_mpg       = auto_a$mpg - auto_b$mpg,
    dif_cilindros = auto_a$cilindros - auto_b$cilindros
  )
}

# b)
a1 <- list(mpg = mtcars$mpg[1], cilindros = mtcars$cyl[1])
a2 <- list(mpg = mtcars$mpg[2], cilindros = mtcars$cyl[2])
comparar_autos(a1, a2)
# $dif_mpg: 0   $dif_cilindros: 0  (los dos primeros Mazdas son iguales)

# c)
comparar_autos(list(mpg = 20), list(consumo = 25))
# $dif_mpg: NA  (auto_b$mpg es NULL, NULL - 20 da numeric(0), envuelto en NA)
# $dif_cilindros: NA  (ambos campos faltan)

Para una versión defensiva: validá los nombres con stopifnot(c("mpg", "cilindros") %in% names(auto_a)). Lo vamos a ver en la guía 4.

  3. Mostrá los tres con menor mpg.

Ejercicio 14: aplicar resumen_min a varias columnas

  1. Aplicá resumen_min por separado a las columnas mpg, hp y wt de mtcars. Guardá los resultados en una lista nombrada.

  2. Hacé lo mismo en una sola línea con lapply. Pista: lapply(mtcars[, c("mpg", "hp", "wt")], resumen_min).

  3. ¿Qué tipo devuelve lapply y qué devuelve sapply? Compará los resultados.

TipSolución 
# a)
resumenes <- list(
  mpg = resumen_min(mtcars$mpg),
  hp  = resumen_min(mtcars$hp),
  wt  = resumen_min(mtcars$wt)
)

# b)
resumenes_v2 <- lapply(mtcars[, c("mpg", "hp", "wt")], resumen_min)
identical(resumenes, resumenes_v2)   # TRUE

# c)
class(resumenes_v2)                                       # "list"
class(sapply(mtcars[, c("mpg", "hp", "wt")], resumen_min))  # "matrix" (4x3)

Cuándo cada uno: lapply siempre devuelve lista (predecible). sapply intenta simplificar a vector o matriz si todos los elementos tienen la misma forma. Para código robusto, lapply es más seguro.

Ejercicio 15: tu propio data.frame

  1. Creá un data.frame con tres pacientes y cuatro columnas: id (integer), edad (double), peso_kg (double), fuma (logical).

  2. Aplicá resumen_min a la columna edad.

  3. Aplicá summary() al data.frame completo. Compará la salida con la de resumen_min.

TipSolución 
# a)
pacientes <- data.frame(
  id      = c(1L, 2L, 3L),
  edad    = c(34, 56, 47),
  peso_kg = c(72.5, 88.0, 65.3),
  fuma    = c(FALSE, TRUE, FALSE)
)

# b)
resumen_min(pacientes$edad)
# media: 45.67  mediana: 47  min: 34  max: 56

# c)
summary(pacientes)
# para columnas numericas: min/Q1/median/mean/Q3/max
# para fuma (logical): cuenta de TRUE y FALSE
# para id (integer): igual que numericas

Diferencia con resumen_min: summary hace una cosa distinta para cada tipo de columna (numérico, factor, lógico, character). Es un caso clásico de dispatch de método sobre clases en R: la función real es summary.data.frame, que llama a summary.numeric, summary.logical, etc.

Ejercicio 16: percentiles con quantile

  1. Aplicá quantile(mtcars$mpg) (sin argumentos extra). Identificá el percentil 25, la mediana y el percentil 75.

  2. Aplicá quantile(mtcars$mpg, probs = c(0.1, 0.5, 0.9)). ¿Qué representa cada valor?

  3. Definí rango_intercuartilico(x) usando quantile. Aplicalo a mtcars$mpg y compará con IQR(mtcars$mpg).

TipSolución 
# a)
quantile(mtcars$mpg)
# 0%    25%   50%   75%   100%
# 10.4  15.4  19.2  22.8  33.9
# Q1 = 15.4, mediana = 19.2, Q3 = 22.8

# b)
quantile(mtcars$mpg, probs = c(0.1, 0.5, 0.9))
# 10%   50%   90%
# 14.3  19.2  30.1
# 10% de los autos tienen mpg <= 14.3; 90% tienen mpg <= 30.1

# c)
rango_intercuartilico <- function(x) {
  q <- quantile(x, probs = c(0.25, 0.75))
  unname(q[2] - q[1])
}
rango_intercuartilico(mtcars$mpg)   # 7.375
IQR(mtcars$mpg)                      # 7.375

Curiosidad: quantile admite 9 algoritmos distintos para interpolar entre puntos cuando el percentil cae entre dos observaciones (type = 1, 2, ..., 9). Por defecto usa type = 7. Diferentes paquetes y libros usan tipos distintos, por eso a veces los resultados no coinciden exactamente.

Ejercicio 17: correlación con cor

  1. Calculá cor(mtcars$mpg, mtcars$hp). Interpretá el signo y la magnitud (cor toma valores entre -1 y 1).

  2. Calculá la matriz de correlaciones entre mpg, hp, wt y cyl con cor(mtcars[, c("mpg", "hp", "wt", "cyl")]).

  3. Identificá el par de variables más correlacionado y el menos correlacionado.

TipSolución 
# a)
cor(mtcars$mpg, mtcars$hp)   # -0.776
# correlacion fuerte y negativa: mas potencia, menos consumo eficiente

# b)
M <- cor(mtcars[, c("mpg", "hp", "wt", "cyl")])
# todas negativas con mpg, fuertes positivas entre hp/wt/cyl

# c)
M[upper.tri(M)]                       # solo el triangulo superior
abs_off <- abs(M); diag(abs_off) <- 0  # ignorar la diagonal (1s)
which(abs_off == max(abs_off), arr.ind = TRUE)
# par mas correlacionado: cyl-hp (~0.83) o similar

Curiosidad: cor por defecto usa Pearson (lineal). Para relaciones no lineales monotónicas (como rangos) usá cor(..., method = "spearman") o "kendall". Pearson puede ser engañoso si los datos no son aproximadamente lineales.

Ejercicio 18: leer la documentación

  1. Abrí ?sapply. Identificá los argumentos X, FUN y simplify.

  2. Probá sapply(1:5, function(i) i^2). ¿Devuelve un vector o una lista?

  3. Probá lapply(1:5, function(i) i^2). Compará con la salida de sapply y explicá cuándo conviene cada uno.

TipSolución 
# a)
?sapply
# X: el vector/lista a recorrer
# FUN: la funcion a aplicar a cada elemento
# simplify: si TRUE (default), simplifica a vector/matriz si puede

# b)
sapply(1:5, function(i) i^2)
# 1 4 9 16 25 (vector numerico)

# c)
lapply(1:5, function(i) i^2)
# lista de 5 elementos: [[1]] 1, [[2]] 4, ..., [[5]] 25

Cuándo cada uno:

  • Si todos los resultados tienen la misma forma (escalar o vector de igual largo): sapply para vector/matriz simplificada.
  • Si los resultados pueden tener formas distintas, o querés garantía de tipo: lapply siempre devuelve lista.
  • Para código robusto, vapply(X, FUN, FUN.VALUE = ...) exige una forma específica y falla temprano si algo no la cumple.

Ejercicio 19: la caja de herramientas sobre datos sintéticos

  1. Generá un vector x con rnorm(100, mean = 50, sd = 10) (100 valores normales). Para detalles, ?rnorm.

  2. Aplicale resumen_min, dif_promedio y dentro_rango con a = 40, b = 60.

  3. ¿Aproximadamente cuántos valores caen en [40, 60]? Compará con la regla 68-95-99.7 de la distribución normal (esperado: ~68%).

TipSolución 
set.seed(42)   # opcional, para reproducir el ejemplo
x <- rnorm(100, mean = 50, sd = 10)

# b)
resumen_min(x)   # media ~50, mediana ~50, rango ~25-75
hist(dif_promedio(x))   # debe verse centrada en 0
sum(dentro_rango(x, 40, 60))   # ~68

# c)
prop <- mean(dentro_rango(x, 40, 60))
prop   # 0.65 a 0.72 segun la semilla
# La regla 68-95-99.7 predice 68% para [mu - sigma, mu + sigma].

Curiosidad: la regla 68-95-99.7 es exacta solo en el límite (n → ∞). Con n = 100 hay variabilidad de muestra. Si repetís muchas veces y promediás, vas a converger al 68.27% exacto: pnorm(1) - pnorm(-1).

Ejercicio 20: buenos nombres

  1. Tomá este snippet:

    f <- function(x, y) x + y * 0.6213
a <- 100
print(f(0, a))

    Pista: el factor 0.6213 convierte kilómetros a millas.

  2. Reescribilo cambiando los nombres para que la intención sea clara sin necesidad de comentarios. Pensá nombres descriptivos para f, a, x, y.

  3. Justificá tu elección en una línea.

TipSolución 
# Reescritura
km_a_mi <- function(millas_base, kilometros) {
  millas_base + kilometros * 0.6213
}

distancia_km <- 100
print(km_a_mi(0, distancia_km))   # 62.13

Justificación: el nombre de la función dice qué convierte y los parámetros indican qué cantidad va en cada lugar (millas_base parte de un offset, kilometros es lo que se convierte). Quien lee el código entiende sin pista.
